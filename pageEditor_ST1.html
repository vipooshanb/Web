Splash Page

Technical Description

HTML Techniques:-

Used HTML techniques like meta refresh tag to redirect users to the Home page in 4 seconds. Used section and div tag to struture the content. Used h1 for the heading, p for the description. Used img tag to enter the image of the splash screen. implemented an Animated loader within a div tagto provide a visual cue for the users while they wait.

CSS Techniques:-

The CSS styles the Aqua Verse splash screen with modern, responsive design and animations. A global reset ensures consistency, and Flexbox centers content. The background uses a dimmed full-screen image. Keyframe animations like fadeIn, slideDown, bounce, and load create smooth transitions, a pulsating loader, and a dynamic progress bar. Typography and responsive layouts ensure the splash screen looks engaging across all devices.

Accessibility

The Aqua Verse splash screen uses semantic HTML for better accessibility, with headings, descriptive text, and alt attributes for images. An animated loader includes ARIA roles to support assistive technologies, while a meta refresh tag redirects users after 4 seconds. CSS ensures a responsive, high-contrast design with smooth animations and a prefers-reduced-motion option for users sensitive to movement, creating an engaging and accessible experience across all devices.

Volunteer Page

Technical Description

HTML Techniques:-

The HTML page structures a volunteer opportunities platform for “AquaVerse” using semantic elements and responsive design. It includes a hero section with headings, descriptions, and a call-to-action button. Volunteer programs are displayed as dynamic cards in a grid layout with images, ratings, and action buttons. Dropdown filters, a feedback form with emoji ratings, and a scrollable reviews section enhance interactivity. Font Awesome icons and external stylesheets are integrated for a modern and engaging user interface.

CSS Techniques:-

This CSS design uses modern techniques like Google Fonts for clean typography, a global reset for consistency, and flexbox/grid for responsive layouts. Background images, gradients, and shadows add depth, while hover effects, transitions, and keyframe animations enhance interactivity. Features like scrollable sections, emoji ratings, and media queries ensure a dynamic, mobile-friendly, and user-focused UI.

Accessibility

The AquaVerse volunteer platform uses semantic HTML for better accessibility, with clear headings, alt text for images, and ARIA labels for interactive elements like filters and forms. Font Awesome icons include aria-hidden attributes for screen readers, and the feedback form supports keyboard navigation. The CSS ensures high contrast for readability, responsive layouts for all devices, and includes prefers-reduced-motion for users sensitive to animations. Together, these techniques create an interactive, mobile-friendly, and accessible user experience.

Content Page

Technical Description

HTML Techniques:-

The webpage uses semantic HTML5 with a structured layout featuring a hero section, semantic <section> elements for threats, and an <aside> sidebar with a table of contents and ocean fact card. Font Awesome icons enhance visuals, and anchor links with a “Back to Top” button improve navigation. External CSS ensures responsive and interactive design.

CSS Techniques:-

The CSS uses modern techniques like custom properties for easy theming, Flexbox for alignment, and gradients for vibrant backgrounds. Hover effects, transitions, and animations make elements interactive, while shadows add depth. Media queries ensure responsiveness on all screens, and fixed elements like the “Back to Top” button enhance usability, creating a clean and user-friendly design.

Accessibility

The webpage uses semantic HTML5 for better accessibility, with clear headings, alt text for images, and ARIA labels for interactive elements like the sidebar and navigation buttons. Anchor links and the “Back to Top” button support keyboard navigation. Font Awesome icons are marked with aria-hidden where needed for screen readers. The CSS ensures high contrast for readability, responsive layouts for all devices, and includes prefers-reduced-motion to assist users sensitive to animations. Together, these techniques create an interactive, mobile-friendly, and accessible user experience.

Challenges and Lessons Learned

During the development of these web pages, we encountered several challenges that required collaborative problem-solving and adaptation. The primary hurdles involved:

  • Image Acquisition and Optimization: Securing appropriate, high-quality images proved more time-consuming than initially anticipated. Finding images that accurately represented our content while remaining freely available for use (or acquiring the necessary licenses) required significant research and effort. We primarily used Adobe Stock (or specify which Adobe service) for downloading images, but initial image downloads often resulted in overly large file sizes that negatively impacted page load speed.


Resolution and Lessons Learned:

  • Image Optimization: To address the image size issue, we implemented a multi-pronged approach. We utilized CloudConvert to convert JPG and PNG images to the WebP format for improved compression and quality. We also optimized images by resizing them to appropriate dimensions for their intended display size, and strategically compressed images using tools like TinyPNG and ImageOptim (or specify what you used if you used alternatives). The key lesson here was to prioritize image optimization from the outset of the project, rather than addressing it as an afterthought.
